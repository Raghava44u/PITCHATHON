👩 Workflow 01
Women Safety Mode
Self-driven · Continuous monitoring · Auto emergency trigger
flutter_background_service
Supabase Realtime
FCM + Native SMS
01
📱 App Launch + Auth
User opens Threenetra AI. Supabase anonymous or phone auth. Mode selection screen appears. User selects Women Safety Mode.
supabase_auth flutter_dotenv riverpod
02
🛡️ Permission Grant
System requests: Location (foreground + background), Microphone, SMS, Contacts. Uses permission_handler. Graceful degradation if partial permissions granted.
location microphone sms contacts
03
⚙️ Add Trusted Contacts
User adds up to 5 trusted contacts (phone numbers). Stored in Supabase contacts table linked to user_id. Contacts receive SMS + FCM on emergency.
04
▶️ Activate Safety Mode
User taps the large Activate button. Background service starts (flutter_background_service). GPS polling begins every 30s. Motion sensor stream opens via sensors_plus. Risk evaluation loop initializes.
background_service geolocator sensors_plus
05
📡 Continuous Monitoring Loop
Every 30 seconds: collect GPS coordinates + speed, read accelerometer variance, check current hour (isNight if 21:00–05:00), fetch crowd score from Supabase heatmap, pass all data to RiskEngine.evaluate(). Location logged to Supabase location_logs.
RiskEngine LocationService MotionService
06
⚡ Risk Level Output
Risk Engine returns LOW / MEDIUM / HIGH with a human-readable reason string (explainability). Dashboard updates in real-time via Riverpod state. AI tip card fetched from Gemini free tier when MEDIUM+.
● Low Risk
Continue & Monitor
Loop continues every 30s. Green status on dashboard. No alerts sent.
● Medium Risk
Warn + AI Tip
Push notification to user. Amber dashboard. AI safety tip card shown. Contacts NOT alerted yet.
● High Risk
Emergency Trigger
Immediate auto-trigger of full emergency action engine. No user action needed.
🆘 Emergency Action Engine (HIGH Risk)
Send SOS SMS + maps.google.com location link to all trusted contacts
Start audio recording in background (AudioService, saved as .m4a locally)
Send FCM push notification to trusted contacts' devices
Log alert to Supabase alerts table with lat, lng, reason, audio_url
Optional: trigger phone call intent to primary contact
Works fully offline — SMS fires even without internet connection
07
🤫 Shake-to-SOS (Hackathon Feature)
Accelerometer detects 3× sharp shake pattern from lock screen. Immediately fires emergency engine without unlocking phone. Implemented via background service accelerometer stream + debounce logic.
MotionService sensors_plus background_isolate
08
✅ Resolution + Safety Journal
User marks alert resolved. Audio uploaded to Supabase Storage if internet available. Incident auto-logged to Safety Journal with timestamp, location, reason. Exportable as PDF for legal use.
supabase_storage incident_log pdf_export
🧒 Workflow 02
Child Safety Mode
Parent-controlled · Geo-fencing · Live tracking · Inactivity detection
Supabase Realtime WS
OpenStreetMap / flutter_map
GeoFenceService
01
👨‍👩‍👧 Parent Setup (One-time)
Parent installs Threenetra on their device, selects Child Safety Mode. Creates child profile. Installs Threenetra on child's device and links via a 6-digit pairing code stored in Supabase.
users_table pairing_code supabase_auth
02
🗺️ Draw Safe Zones (Geo-fences)
Parent opens Parent Dashboard (OSM map). Taps to draw geo-fence circles (e.g. home radius 200m, school 150m). Zones saved to Supabase geofences table. Active/inactive toggle per zone. Displayed as filled polygons on flutter_map.
GeoFenceService flutter_map openstreetmap
03
📡 Child Device: Live Tracking
Child's phone runs background GPS every 30s. Location pushed to Supabase Realtime channel. Parent dashboard subscribes to realtime stream — child's location marker updates live on OSM map without refresh.
location_logs supabase_realtime websocket
04
🔍 Geo-fence Breach Check
On each GPS update, GeoFenceService computes Haversine distance from child's position to each active zone center. If distance > zone radius → breach detected. Breach creates an alert record with child_user_id, zone_id, lat, lng.
haversine GeoFenceService alerts_table
05
😴 Inactivity Detection
Motion service monitors accelerometer variance on child's device. If child has not moved significantly for >30 min AND is not in a known safe zone → Risk Engine returns MEDIUM with reason "Child inactive too long". Parent alerted immediately.
MotionService RiskEngine threshold_30min
● Inside Zone
All Safe
Child is inside a safe zone and active. Green status on parent dashboard.
● Inactivity / Unknown Area
Alert Parent
FCM push to parent. "Check on your child" notification with last known location.
● Geo-fence Breach
Immediate Alert
FCM push + SMS to parent. Deeplink opens Parent Dashboard to exact breach point on map.
06
🔔 Parent Receives Alert
FCM push with data payload: child name, alert type, coordinates. Tapping notification deep-links to Parent Dashboard map view. Community safety heatmap overlay shows crowd safety of breach area. Supabase Edge Function triggers FCM automatically on new HIGH alert insert.
FCM EdgeFunction deep_link heatmap
07
📊 Activity Feed + History
Parent dashboard shows full child activity timeline: check-ins, movement patterns, alerts, zone entries/exits. All persisted in Supabase alerts + location_logs. Exportable for school/guardian records.
⚡ Workflow 03
Risk Engine Pipeline
Rule-based v1 · TFLite-ready v2 · Explainable output · Both modes feed this
Pure Dart module
TFLite slot ready
Gemini AI tips
01
📥 Input Collection (RiskInput)
Background service assembles a RiskInput object every 30 seconds containing: lat, lng, speed (m/s), currentHour (0–23), motionVariance (from accelerometer), crowdScore (0.0–1.0 from heatmap), inactivityMinutes, appMode (women | child).
LocationService MotionService CrowdSafetyService RiskInput
02
🧠 Rule Evaluation (v1 — Active)
Rules evaluated in priority order:

IF isNight AND speed > 15 → HIGH "Fast movement at night"
IF crowdScore < 0.2 → MEDIUM "Isolated area"
IF mode==child AND inactivity > 30min → MEDIUM "Inactive too long"
ELSE → LOW "All clear"
RiskEngine.evaluate() pure_dart unit_testable
03
🤖 TFLite Slot (v2 — Future)
The evaluate() method body is swappable. v2 replaces rules with TFLite model inference. Input tensor: [lat, lng, speed, hour, crowdScore, motionVariance]. Output tensor: [riskLevel(0-2), confidence]. Same RiskResult interface — no upstream code changes needed.
tflite_flutter model.tflite same_interface
04
📤 RiskResult Output
Returns a typed RiskResult(level: RiskLevel.HIGH, reason: "..."). Reason string shown on dashboard for explainability and transparency. Riverpod provider notifies UI. AlertEngine.dispatch(result) called next.
05
💡 AI Tip Fetch (MEDIUM+)
If risk ≥ MEDIUM, AITipService sends a context-aware prompt to Gemini free tier API: mode, time, crowdScore, and risk reason are included. Response parsed and displayed as a non-intrusive card. Cached locally to avoid repeat API calls for same context. Falls back to a local tip bank if offline.
AITipService gemini_free_api hive_cache offline_fallback
● Low
Silent Continue
No notifications. Dashboard stays green. Loop repeats in 30s.
● Medium
Warn + AI Tip
Local push + dashboard amber. AI tip card. No emergency actions yet.
● High
Emergency Engine
Full SOS chain fires: SMS, FCM, audio record, Supabase log.
06
🗺️ Community Heatmap Feed-in
CrowdSafetyService converts current GPS to H3 hex index (resolution 9). Queries Supabase safety_ratings table for average rating of that hex. Normalizes to 0.0–1.0 → crowdScore. This directly feeds into Risk Engine as an input, creating a community-powered risk signal.
H3_hex_grid CrowdSafetyService anonymous_ratings
07
📋 Audit + Safety Journal
Every RiskResult ≥ MEDIUM is persisted to Supabase alerts with full context: level, reason, lat, lng, timestamp, audio_url. Users view full incident timeline in Safety Journal. Exportable as PDF for police reports or legal evidence. Loop restarts in 30s.
alerts_table incident_log pdf_export loop_30s
